Appendix A: Project Information (What we've built)

Overview: Prashna-AI is an adaptive, AI-driven learning platform designed to generate dynamic quizzes from static uploaded content (text, PDFs, or URLs). The core objective is to move away from static testing and instead test students adaptively, matching the difficulty of the questions to their proven skill level using an Elo rating system.

Key Achievements:

  • Developed a fully functioning full-stack application from scratch.
  • Integrated a real-time Elo rating system to track user skill.
  • Built a multi-provider fallback AI chain (OpenAI -> Groq -> Ollama) to ensure high uptime for quiz generation.
  • Implemented an intelligent content processing pipeline that chunks data, analyzes complexity, discovers topics (via NLP), and summarizes.
  • Handled advanced UI state management, real-time feedback, and secure, role-based admin routing.
  • Optimized massive database retrievals, eliminating N+1 query loops to make the dashboard load instantly.

Appendix B: Feature Breakdown

Frontend Features (React + Vite)

  • Responsive, Modern UI: Clean, animated interface with a seamless drag-and-drop file upload, custom toggle switches, and a global Light/Dark mode.
  • Adaptive Quiz Interface: Real-time progress bars, interactive option highlighting, and a lenient short-answer grader that understands context rather than punishing typos.
  • Dynamic Dashboard & Analytics: Renders visual data including Elo tracking, Weak Area tagging, and performance prediction.
  • Role-Based Access Control (RBAC): Secure routing that completely hides and blocks the Admin Panel from non-developer accounts.

Backend Features (FastAPI + Python)

  • Advanced NLP Pipeline: Uses spaCy to extract entities and text complexity (textstat), and BERTopic / frequency analysis to discover core concepts in uploaded text.
  • Adaptive Engine: Calculates standard Elo Rating updates instantly after every quiz, tracking historical correctness and response times.
  • Multi-Agent LLM Generation: Sends targeted prompts using specific cognitive levels from Bloom's Taxonomy to generate tailored questions. Uses background tasks to prevent the server from hanging during intense processing.
  • Supabase Integration & Optimization: Handles authentication natively, and uses precisely tailored bulk SQL/REST queries to fetch hundreds of attempts in a fraction of a second without event-loop blocking.

Appendix C: Potential Q&A with Your Coordinator

How does the Adaptive Mode actually work?
"When enabled, the system reads the user's current Elo Rating. If the Elo is low (<800), it forces the AI to generate 'Easy' questions focused on basic recall. If it's high (>1200), it forces 'Hard' questions focused on analysis and evaluation, based on Bloom's Taxonomy. The middle tier gets 'Medium' questions."
Why did you choose FastAPI for the backend?
"FastAPI is incredibly fast because it's built on Starlette and Pydantic. It natively supports asynchronous programming, which is crucial for an AI app where we spend a lot of time waiting for LLM network responses. It prevents the server from blocking other users."
How are you managing the AI generation if the API goes down or is slow?
"We implemented a Multi-Provider Fallback Chain. The system tries OpenAI first. If it times out or fails, it automatically falls back to Groq (which is exceptionally fast), and can even fall back to a local Ollama instance if necessary, ensuring constant uptime."
How do you handle the grading for Short Answer text questions?
"We implemented a lenient string-matching algorithm. Instead of demanding an exact character-to-character match, the grader splits both the correct answer and the user's answer into core keywords (ignoring stop words). If there is a sufficient intersection of keywords or a substring match, it marks the answer as correct. It understands the concept rather than just the exact spelling."
Did you run into any major performance issues, and how did you solve them?
"Yes, initially the Dashboard took very long to load because of an 'N+1 Query Loop'—the backend was pinging the database separately for every single quiz a user had ever taken. I refactored the data layer to fetch all attempts in a single massive bulk query, reducing the load time to under a second. I also had to convert async routes to standard thread-based routes to allow FastAPI to handle synchronous database calls efficiently."
How is the Admin Panel secured?
"It uses strict Role-Based Access Control both on the frontend and backend. The React frontend will bounce any unauthorized user back to the dashboard if they manually enter the URL. Even if they bypass the UI and hit the API directly, the FastAPI backend checks the JWT token's email payload and strictly returns a '403 Forbidden' error if the email doesn't manually match the developer account."
